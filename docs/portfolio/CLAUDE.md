## 작업 규칙

### 문서 작성 시
- **수치 기반**: 정량적 성과 (%, 건수, 시간 단축) 우선
- **간결하게**: 불필요한 수식어 제거
- **마크다운**: 표, 코드블록 적극 활용
- **구조화**: 개요 → 기술스택 → 담당업무 → 성과 → 도전과제

### 파일 네이밍
- 메인 문서: `README.md`
- 세부 문서: `{주제}.md` (kebab-case)
- 이미지: `{프로젝트명}-{설명}.png`

### 새 회사 지원 시
1. `applications/{회사명}/` 폴더 생성
2. `README.md`에 직무 분석 + 내 경험 연결점 정리

### 새 경력 추가 시
1. `experience/{회사명}/` 폴더 생성
2. `README.md`에 회사 개요 + 담당 업무 정리
3. 프로젝트별 상세 문서 추가

## n8n 워크플로우 상세

### 핵심 워크플로우 목록

1. **엑셀 주문 처리** - 7단계 우선순위 매칭
2. **문자/이미지 주문 처리** - OCR → JSON → 매칭
3. **외부주문 알림톡 + 네이버 스토어 발송처리**
4. **DB 캐시 스케줄링** - 대용량 데이터 n8n 테이블 노드로 캐시
5. **매입/일반 주문 처리**

### 7단계 우선순위 매칭 시스템

```
1순위: 동일 거래처 정확 일치 (confidence: 1.0)
2순위: 타 거래처 정확 일치 (confidence: 0.9)
3순위: 동일 거래처 부분 포함 (confidence: 0.8)
4순위: 타 거래처 부분 포함 (confidence: 0.7)
5순위: 정규화 후 일치 (confidence: 0.6)
6순위: 유사어 그룹 매칭 (confidence: 0.5)
7순위: AI 처리 필요 → 관리자 수동 입력 → 자연어 사전 등록
```

### 매칭 시스템 발전 과정

| 단계 | 개선 내용 | 매칭률 |
|------|----------|--------|
| 초기 | 정확 일치만 | ~50% |
| 1차 | AI로 자연어 → 보편적 식재료명 변환 | ~60% |
| 2차 | 최근 주문 기반 매칭 추가 | ~75% |
| 3차 | 작업자 매칭 이력 DB 저장 | ~85% |
| 4차 | n8n 테이블 노드 캐시화 (DB 부하 감소) | ~90% |
| 5차 | 자연어 사전 구축 + 7단계 우선순위 | 96-97% |

### Google Calendar 연동
- 작업 완료 시 자동으로 캘린더에 등록
- 리포트 URL, 통계 (매칭률, 처리 건수) 포함
- 작업 이력 추적 및 성과 확인 용이

## 기술적 도전 과제

### 1. 자연어 → ERP 품목 매칭
- **문제**: 고객 주문서의 품목명과 ERP 품목명 불일치
- **예시**: "소 국거리" → "한우 국거리용 (1kg)"
- **해결**: 7단계 우선순위 매칭 + 자연어 사전 학습 시스템
- **결과**: 50% → 97% 매칭률 달성

### 2. 다채널 입력 통합
- **문제**: 엑셀, 문자, 이미지 등 다양한 형식의 주문 처리
- **해결**: 프론트에서 JSON 표준화 → n8n 단일 워크플로우로 처리
- **결과**: 어떤 채널이든 동일한 매칭 로직 적용

### 3. DB 부하 최적화
- **문제**: 매 요청마다 전체 품목 DB 조회 시 성능 저하
- **해결**: n8n 테이블 노드로 캐시, 스케줄링으로 주기적 갱신
- **결과**: 응답 속도 개선, DB 부하 감소

### 4. 팀 협업 환경
- **문제**: 개발 경험 부족한 팀원들과 협업
- **해결**: 프로젝트 구조화, 백업 자동화, Claude 스킬로 반복 작업 자동화
- **결과**: 팀 전체 개발 생산성 향상

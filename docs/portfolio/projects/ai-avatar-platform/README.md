# AI 아바타 인사이트 플랫폼

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | AI 아바타 인사이트 플랫폼 |
| 기간 | 2025.01 ~ 2025.05 |
| 프로그램 | 메타버스 SW 아카데미 |
| 팀 구성 | 4인 (1기획, 3개발) |
| 역할 | 서비스 기획 + 백엔드 거의 전체 개발 + AI 오케스트레이션 + n8n 워크플로우 |
| 수상 | 메타버스 SW 아카데미 **최우수팀** |
| GitHub | [abora_front](https://github.com/jhon829/abora_front), [abora_backend](https://github.com/jhon829/abora_backend) |

### 한 줄 소개
**다중 AI 아바타**가 사용자의 아이디어에 대해 서로 대화하며 다양한 관점의 인사이트를 제공하는 대화형 브레인스토밍 도구

---

## 기술 스택

```yaml
Backend: FastAPI, Python
AI Framework: AutoGen
AI APIs: OpenAI API, Gemini API, Perplexity Sonar API, Whisper
3D Rendering: Blender, React Three Fiber
Workflow: n8n (질문 분류 → 전문가 라우팅)
```

---

## 담당 업무

### 서비스 기획
- **플랫폼 핵심 컨셉**: 사용자가 질문하면 다양한 페르소나의 AI가 상호 대화하며 문제를 다각도로 분석
- **3단계 대화 구조** 기획: 사실 확인 → 가정 검증 → 패러다임 전환
- AI 캐릭터 페르소나 설정 (분석형/직관형, 각각 다른 AI 모델 기반)
- 사용자 흐름 설계: 아이디어 입력 → 질문 강화 → 키워드 추출 → 시장분석 → AI 대화

### 백엔드 개발 (거의 전체)
백엔드 코드의 대부분을 직접 작성:
- **logic.py** (291줄): AI 오케스트레이션 핵심 로직, AutoGen 기반 다중 에이전트 대화 제어
- **sonar.py** (140줄): Perplexity Sonar API 연동, 키워드 분류/추출, 시장 분석 전처리
- **auth, user, chatsession 모듈**: 인증, 사용자 관리, 대화 세션 관리

### AI 오케스트레이션 시스템 개발
- OpenAI, Gemini 등 다양한 AI 모델을 AutoGen으로 연결
- **순차적 대화 흐름을 제어하는 시스템** 구축
- AI 아바타 간 **발화 규칙 및 컨텍스트 전달 방식** 정의 및 구현
- Sonar API로 **전략 요약, 키워드 추출, 시장 분석 전처리**
- AI 대화 결과를 **텍스트 + 3D 인터랙션으로 시각화**

### n8n 워크플로우
- 질문 분류 → 전문가 라우팅 시스템 (developer / designer / planner)
- 사용자 입력을 분석하여 적절한 전문가 AI에게 자동 라우팅
- 별도 데모로 구현 (핵심 AI 오케스트레이션과는 분리)

## AI 캐릭터 페르소나

실제 코드에 구현된 페르소나:

| 페르소나 | 코드명 | AI 모델 | 특징 |
|---------|--------|---------|------|
| 분석형 | 상균 | GPT 기반 | 데이터 기반 논리적 분석 |
| 직관형 | 채영 | Gemini 기반 | 창의적 관점, 아이디어 발산 |

### 3단계 대화 구조
1. **사실 확인**: 주어진 정보의 객관적 검증
2. **가정 검증**: 숨겨진 가정과 전제 탐색
3. **패러다임 전환**: 새로운 관점과 가능성 도출

## 주요 성과

- 메타버스 SW 아카데미 **최우수팀** 수상
- 복수 AI를 활용한 협업형 챗봇 구조 설계를 통해 **AI를 '도구'가 아닌 '파트너'로 활용하는 서비스 모델** 기획 경험 확보
- 대화형 UX 설계와 기술 로직 간의 연결을 주도하며 **기획-기술 중간지점에 대한 깊은 이해력** 향상
- LLM 기반 API 통신 구조와 AutoGen 라이브러리를 활용한 **실제 오케스트레이션 경험**

## 인터뷰 기반 상세 기록

### 타겟 구체화 과정
- **초기**: 막연하게 "아이디어 검증 도구"로 시작
- **구체화**: 초기 창업가 / 마케터를 타겟으로 설정
- **검증**: 실제 인터뷰를 통해 타겟 적합성 확인
- **배운 점**: 타겟을 구체적으로 좁힐수록 서비스 방향이 명확해짐

### AI 대화 통제의 어려움과 해결
- **문제**: AI 캐릭터들이 대화 중 주제를 벗어나거나 같은 말을 반복하는 현상
- **시도**: 리액션 + 질문 구조로 대화 흐름 제어 시도
- **해결**: 3단계 대화 구조 (사실 확인 → 가정 검증 → 패러다임 전환)로 대화 프레임을 강제
- **결과**: AI가 단계별로 역할에 맞는 발화를 하도록 유도, 대화 품질 향상

### 팀 소통 방법
- **스토리보드 활용**: 기획 의도를 프론트엔드 개발자에게 전달할 때 직접 스토리보드를 그려서 소통
- 텍스트 기반 설명보다 시각적 전달이 훨씬 효과적이었음
- 기획자-개발자 간 인식 차이를 줄이는 데 큰 도움

### 기획-기술 타협 경험
- **원래 기획**: AI 아바타가 다양한 리액션(표정, 제스처 등)을 표현
- **기술적 한계**: 3D 아바타의 다양한 리액션 구현이 시간/기술적으로 어려움
- **타협안**: 채팅 중 **시선 추적**으로 대체 — 사용자가 아바타를 바라보면 반응하는 방식
- **배운 점**: 기획 의도를 100% 구현하기 어려울 때, 핵심 UX를 살리면서 기술적으로 실현 가능한 대안을 찾는 과정이 중요

### 가장 어려웠던 점
- AI 모델마다 응답 패턴이 달라서 **일관된 대화 흐름을 만드는 것**이 가장 도전적
- 특히 오픈소스 모델과 상용 모델의 품질 차이를 프롬프트 엔지니어링으로 메우는 작업

## 배운 점 (인터뷰 기반)

### 1. 타겟 구체화의 중요성
> "타겟을 구체적으로 좁히니까 서비스 방향이 확실해졌다"

- 막연한 "모두를 위한 서비스" → 초기 창업가/마케터로 구체화
- 타겟이 명확해지자 기능 우선순위도 자연스럽게 결정

### 2. 스토리보드를 통한 소통
- 기획 의도를 말이나 글로 전달하면 해석이 달라짐
- 직접 그려서 보여주니 팀원들과 한 번에 합의 가능

### 3. AI 대화 통제 기술
- LLM은 자유도가 높아서 원하는 방향으로 유도하기 어려움
- 대화 구조(프레임)를 강제하는 것이 프롬프트 튜닝보다 효과적

### 4. 기획과 기술의 타협
- 기획 의도를 100% 구현할 수 없을 때, **핵심 UX를 살리는 대안**을 찾는 능력
- 리액션 → 시선추적으로의 전환처럼, 사용자 경험의 본질은 유지하면서 기술적으로 가능한 방법 선택

## 아쉬운 점

- **실사용자 테스트 미진행**: 아카데미 기간 내에 실제 창업가/마케터 대상 테스트를 하지 못함
- **아바타 리액션 한계**: 기획했던 다양한 표정/제스처를 시선추적으로 대체한 부분
- **대화 품질 일관성**: AI 모델 간 응답 품질 편차를 완전히 해소하지 못함

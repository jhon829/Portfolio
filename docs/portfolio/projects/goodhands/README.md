## 서비스 핵심 컨셉

### 타겟 사용자
> **핵심 인사이트**: 서비스 타겟은 "재외동포 시니어"가 아니라 **"그들의 자녀"**

### 해결하려는 문제
1. **정보 부족**: 해외에 있는 자녀가 한국에 계신 부모님 상태를 알 수 없음
2. **죄책감**: 부모님을 멀리 보내서 느끼는 죄책감
3. **신뢰 부족**: 케어 서비스가 제대로 되고 있는지 확인 불가

### 솔루션
- 케어기버가 매일 체크리스트 + 돌봄노트 작성
- AI가 따뜻한 말투로 리포트 생성
- 해외 자녀에게 실시간 알림 전송

## 데이터베이스 설계 (18개 테이블)

### 사용자 관련
- `users`: 기본 인증 정보
- `caregivers`: 케어기버 프로필
- `guardians`: 가디언 프로필 (해외 자녀)
- `admins`: 관리자
- `seniors`: 시니어 기본 정보
- `senior_diseases`: 시니어 질병 정보

### 요양원 관련
- `nursing_homes`: 요양원 정보

### 돌봄 세션 관련
- `care_sessions`: 돌봄 세션 (출근~퇴근)
- `care_calendar`: 돌봄 일정
- `care_schedules`: 정기 스케줄
- `attendance_logs`: 출퇴근 로그

### 체크리스트 관련
- `checklist_categories`: 체크리스트 카테고리 (영양, 고혈압, 우울증)
- `checklist_questions`: 체크리스트 질문
- `checklist_responses`: 체크리스트 응답
- `checklist_types`: 체크리스트 유형
- `weekly_checklist_scores`: 주간 점수 집계

### 돌봄노트 관련
- `care_notes`: 돌봄노트
- `care_note_questions`: 6개 핵심 질문

### AI/알림 관련
- `ai_reports`: AI 생성 리포트
- `notifications`: 알림
- `feedbacks`: 가디언 피드백

## AI 리포트 시스템 (100% 내 기획/개발)

### 입력 데이터
1. **체크리스트 점수**: 영양상태, 고혈압관리, 우울증 (각 4문항, 4점 척도)
2. **돌봄노트**: 6개 질문 중 랜덤 선택, 케어기버 자유 작성

### 처리 흐름
```
케어기버 퇴근
    ↓
백엔드 → n8n 웹훅 트리거
    ↓
n8n: 체크리스트 데이터 조회 (영양/고혈압/우울증)
    ↓
n8n: 돌봄노트 데이터 조회
    ↓
GPT-3.5: 따뜻한 말투로 리포트 생성
    ↓
백엔드: ai_reports 테이블에 저장
    ↓
가디언에게 알림 발송
```

### GPT 프롬프트 설계 (내가 설계)
```
"당신은 해외에 거주하는 자녀들을 위해 부모님의 돌봄 상황을 
전달하는 따뜻한 케어 전문가입니다.

- 수치적 데이터를 자연스럽고 따뜻한 말투로 변환
- "어머니께서는" 같은 존댓말과 친근한 호칭 사용
- 가족이 읽었을 때 마음이 따뜻해지고 안심할 수 있는 말투

✅ 좋은 예: "할머니께서 요즘 식사를 정말 잘 드시고 계세요."
❌ 피할 표현: "영양상태가 75%에서 87.5%로 개선되었습니다."
```

### 출력 예시
```json
{
  "report_type": "nutrition_report",
  "content": "어머니께서 요즘 식사를 정말 잘 드시고 계세요. 
             지난주보다 드시는 양이 늘어서 영양상태가 많이 좋아지셨어요.",
  "status_code": 1,  // 1:개선, 2:유지, 3:악화
  "checklist_type_code": "nutrition"
}
```

## 체크리스트 시스템

### 카테고리 (3개)
| 카테고리 | 질문 수 | 최대 점수 |
|---------|--------|----------|
| 영양상태 (nutrition) | 2문항 | 8점 |
| 고혈압관리 (hypertension) | 4문항 | 16점 |
| 우울증 (depression) | 4문항 | 16점 |

### 점수 체계 (4점 척도)
- 1점: 매우 안 좋음
- 2점: 안 좋음
- 3점: 보통
- 4점: 좋음

### 트렌드 분석
- 이전 3주간 점수와 비교
- status_code: 1(개선), 2(유지), 3(악화)

---

## 돌봄노트 시스템

### 6개 핵심 질문 (랜덤 선택)
1. **특별한 순간**: 인상 깊었던 순간이나 따뜻했던 에피소드
2. **가족에 대한 그리움**: 가족에 대해 하신 말씀이나 감정
3. **기분과 감정 표현**: 오늘 기분이나 감정 상태
4. **대화 내용과 소통**: 기억에 남는 대화 내용
5. **평소와 다른 변화**: 달라진 점이나 새롭게 관찰된 것
6. **케어 과정 에피소드**: 돌보는 과정에서의 따뜻한 순간

### 데이터 무결성 정책
- **1일 1회 작성 제한**: 복합 UNIQUE KEY
- **수정 불가**: `is_final`, `modification_blocked` 플래그
- PUT/DELETE API → 403 Forbidden 반환

## 기술적 도전 과제

### 1. 첫 백엔드 개발 경험
- **어려움**: 백엔드/프론트 역할 분리 기준이 불명확
- **해결**: FastAPI 구조 학습, API 설계 패턴 습득
- **배움**: 비즈니스 로직은 services/, API 엔드포인트는 routers/로 분리

### 2. 데이터 무결성 보장
- **문제**: 의료 데이터 특성상 수정/삭제 방지 필요
- **해결**: `is_final` 플래그 + API 레벨 차단 (403 반환)
- **결과**: 데이터 무결성 100% 보장

### 3. AI 리포트 말투 설계
- **문제**: 수치 데이터를 가족이 읽기 좋은 형태로 변환
- **해결**: GPT 프롬프트에 구체적 예시 포함, "따뜻한 케어 전문가" 페르소나 설정
- **결과**: 자연스럽고 안심되는 리포트 생성

## 인터뷰 기반 상세 기록

### AI 리포트 기획의 어려움
- **핵심 문제**: "내가 보호자였던 적이 없으니까" — 해외에서 부모님을 케어 맡기는 자녀의 감정을 직접 경험한 적이 없어 리포트 톤/내용 기획이 어려웠음
- **극복 방법**:
  - **관계자 인터뷰**: 실제 케어 현장의 관계자들과 다수 인터뷰 진행
  - **협업기업 도메인 지식**: 인굿핸즈(협업 기업)로부터 재외동포 케어 도메인의 실질적 지식을 전수받음
  - 인터뷰와 도메인 학습을 통해 "자녀가 진짜 알고 싶은 것"과 "안심할 수 있는 말투"를 구체화

### AI 리포트 = 자녀 연락 유도 장치
- AI 리포트의 숨은 설계 의도: 단순 상태 전달이 아니라 **자녀가 부모님께 연락하도록 유도하는 장치**
- 리포트를 읽은 자녀가 "오늘 어머니가 이런 말씀을 하셨구나" → 자연스럽게 전화/영상통화로 이어지도록 설계
- 케어 서비스의 본질: 물리적 돌봄 + **가족 관계 유지**

### 유저 테스트 경험
- **테스트 대상**: 팀원의 할머님과 삼촌
- 만들고 나서 직접 보여드리며 피드백 수집
- 실제 사용자(시니어 + 가족)의 반응을 관찰하며 UX 개선점 도출
- 실제 요양원 테스트는 미진행 (아쉬운 점)

### 가장 중요한 배운 점
- **관계자 인터뷰의 힘**: 직접 경험하지 못한 도메인도 인터뷰를 통해 깊이 있게 이해할 수 있음
- **협업기업의 도메인 지식**: 스타트업과의 협업에서 가장 큰 자산은 그들이 가진 현장 지식
- **기획자의 공감 능력**: 타겟 사용자를 직접 만나보는 것이 데스크 리서치보다 100배 효과적

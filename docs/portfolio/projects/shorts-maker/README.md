## 프로젝트 배경

### 시장 문제 인식

온라인 쇼핑몰 시장에서 소상공인들이 겪는 구조적 문제:

1. **홍보 비용 부담**: 네이버 스마트스토어 등의 홍보는 막대한 비용이 들고 1회성
2. **플랫폼 수수료**: 스마트스토어, 쿠팡 등 이커머스 플랫폼의 높은 수수료 부담
3. **마케팅 인력 부재**: 1인/소규모 사업자는 마케팅 전담 인력을 갖추기 어려움
4. **플랫폼 종속 리스크**: 한 플랫폼에 의존하면 정책 변경 시 매출 타격

### 시장 데이터

- SNS 광고 대행 비용: 광고비의 **15~20%** 추가 대행료 발생
- 숏폼 상품 구매 전환율: 일반 상품 대비 **4배** (W컨셉 기준 6.2%)
- 2026년 이커머스 트렌드: 단일 플랫폼 의존 리스크 증가 → **자체 채널 확보**가 핵심 과제
- 소상공인 마케팅: AI 자동화 + 숏폼이 저비용 대안으로 부상

> **출처**: [이비즈타임즈 2026 이커머스 전망](https://www.e-biztimes.com/news/405458), [카테노이드 숏폼 마케팅](https://blog.catenoid.net/short-form-marketing-importance), [한국경제 숏폼 마케팅 트렌드](https://www.hankyung.com/article/202507161889g)

### 솔루션 방향

> 이커머스 플랫폼에 의존하지 않고, **자사몰로 유입할 수 있는 나만의 채널**을 쇼츠 자동화로 설계해주자

- 개인 동기: 평소 쇼츠를 자주 보며 AI 서비스에 관심 → "쇼츠도 자동화할 수 있겠다"
- 비즈니스 동기: 소상공인의 홍보 문제를 AI 자동화로 해결

## 자동화 파이프라인 (n8n 워크플로우)

**전체 노드 수**: 41개 | **사용 AI 서비스**: 8개 | **상태 관리**: 6단계

```
Webhook (제품 URL 수신)
    ↓
[Stage 1] 제품 정보 수집 · 리서치
    Jina Reader 크롤링 → 데이터 정제 → Perplexity AI 분석
    ↓
[Stage 2] 스크립트 생성
    GPT-4.1-mini (6단계 바이럴 스크립트 공식)
    ↓
[Stage 3] 보이스 생성
    ElevenLabs TTS v3 (via fal.ai) → 타임스탬프 기반 씬 자동 분할
    ↓
[Stage 4] 이미지 생성
    GPT-4.1-mini (프롬프트 생성) → Kie.ai nano-banana-pro (이미지 생성)
    ↓
[Stage 5] 영상 생성
    Kie.ai Seedance 1.5 Pro (Image-to-Video)
    ↓
[Stage 6] 병합
    fal.ai FFmpeg (영상 병합 → 오디오 합성)
    ↓
[Stage 7] 자동 업로드
    YouTube Data API → DB 최종 업데이트
```

### Stage 1: 제품 정보 수집 · 리서치

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 1-1 | Webhook으로 제품 URL, 타겟, 음성, 비율 등 수신 | n8n Webhook |
| 1-2 | 제품 페이지를 마크다운으로 변환하여 크롤링 | **Jina Reader API** |
| 1-3 | 불필요한 내비게이션/팝업 텍스트 제거, 3000자 정제 | JavaScript |
| 1-4 | 제품 분석 JSON 생성 (제품명, 브랜드, 가격대, 셀링포인트, 타겟 고객층, 바이럴 포인트 등 18개 항목) | **Perplexity AI** |

> 크롤링된 데이터만 기반으로 분석하도록 프롬프트에서 명시적으로 제한 (환각 방지)

### Stage 2: 스크립트 생성

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 2-1 | 35~50초 분량(550~700자) 숏폼 나레이션 스크립트 생성 | **OpenAI GPT-4.1-mini** |

**6단계 바이럴 스크립트 공식** (직접 설계):
1. **킬러 후킹** (1.5초) — 스크롤을 멈추게 하는 한 문장
2. **내 이야기** (3~10초) — 공감 유도
3. **경험 전달** (10~25초) — 제품 경험 자연스럽게
4. **의외의 발견** (25~30초) — 반전 포인트
5. **마무리** (25~35초) — 감성적 마무리
6. **자연스러운 CTA** (마지막 3~5초) — 가격 언급은 여기서만 1회

> "대한민국 TOP 숏폼 크리에이터" 페르소나, 반말 통일, 광고 티 제거, 친구에게 카톡 보내는 톤

### Stage 3: 보이스 생성 (TTS)

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 3-1 | 스크립트를 AI 음성으로 변환 (음성: Adam, 속도: 1.2배) | **ElevenLabs TTS v3** (via fal.ai) |
| 3-2 | 문자 단위 타임스탬프를 분석하여 씬(Scene) 자동 분할 | JavaScript (자체 알고리즘) |

**씬 분할 알고리즘** (직접 설계):
- ElevenLabs의 캐릭터 레벨 타임스탬프 활용
- 마침표/물음표/느낌표 기준 문장 분리 → 6~12초 단위로 씬 그룹핑
- 각 씬에 `origin_script`와 `duration` (8초 또는 12초) 자동 부여
- 3초 이하 짧은 씬은 이전 씬에 자동 병합
- 총 duration이 오디오보다 짧으면 8초 씬을 12초로 자동 확장

### Stage 4: 이미지 생성

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 4-1 | 제품 페이지 HTML에서 대표 이미지 URL 추출 | JavaScript |
| 4-2 | 씬별 이미지 생성 프롬프트를 영어로 작성 | **OpenAI GPT-4.1-mini** |
| 4-3 | 각 씬별 이미지 생성 (제품 이미지를 참조 이미지로 활용) | **Kie.ai** (`nano-banana-pro`) |

**이미지 프롬프트 설계** (직접 설계):
- 첫 씬: 문제 상황 (제품 없는 장면)
- 중간 씬: 제품 이미지를 `image_input`으로 전달 (Image-to-Image)
- 마지막 씬: 감성 와이드샷
- 고정 스타일: "Photorealistic, 8K UHD, shot on Sony A7R IV 50mm f/1.4"
- 네거티브 프롬프트: 텍스트, 워터마크, 만화풍 등 제거

### Stage 5: 영상 생성 (Image-to-Video)

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 5-1 | 각 씬의 이미지를 입력으로 동영상 클립 생성 (8초/12초) | **Kie.ai** (`bytedance/seedance-1.5-pro`, Kling) |

> 전체 파이프라인에서 가장 시간이 오래 걸리는 단계 (약 20분 대기)
> 영상에 자체 텍스트 오버레이 금지를 프롬프트로 명시

### Stage 6: 병합

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 6-1 | 여러 영상 클립을 하나의 연속 영상으로 병합 | **fal.ai FFmpeg API** |
| 6-2 | 병합된 영상에 TTS 오디오 트랙 합성 | **fal.ai FFmpeg API** |

> 2단계 합성: (1) 클립 이어붙이기 → (2) 오디오 합성

### Stage 7: 자동 업로드

| 단계 | 동작 | 사용 서비스 |
|------|------|------------|
| 7-1 | 최종 영상 다운로드 | HTTP |
| 7-2 | YouTube에 자동 업로드 (private → 검수 후 공개) | **YouTube Data API** (OAuth2) |
| 7-3 | DB 최종 상태 업데이트 (`complete`) | PostgreSQL (Supabase) |

### 상태 관리 시스템

**DB**: PostgreSQL (Supabase) — `shortmaker.contents` 테이블

| 상태값 | 저장 데이터 |
|--------|------------|
| `research` | Perplexity 분석 결과 JSON |
| `script` | title, description, script, tags |
| `voice` | audio_url (TTS 오디오) |
| `image` | image_urls (씬별 이미지 URL 배열) |
| `video` | video_clips (씬별 영상 URL 배열) |
| `complete` | final_video_url (YouTube Shorts URL) |

> 외부 대시보드에서 각 콘텐츠의 제작 진행 상황을 실시간 추적 가능

## 핵심 챌린지: 시청 유지율 개선

### 문제
쇼츠는 1분짜리 짧은 영상. 조금이라도 지루하면 바로 넘겨버리기 때문에, AI가 생성한 영상이어도 사람이 계속 시청하게 만드는 것이 핵심 과제.

### Before / After (시니어 채널 기준)

## 배운 점

### 1. "어떻게"가 아니라 "무엇을"이 먼저다
> "AI 활용에서 중요한 것은 의도다. 무엇을 AI로 해결해야 할지 고민하는 것이 핵심이다."

- 처음에는 "어떻게 이 기능을 구현할지", "어떻게 자동화할지"에 초점을 맞춤
- 하지만 계속하면서 깨달은 것: **"무엇을 만들지"에 대한 고민이 탄탄하고 오래할수록 품질이 올라간다**
- AI는 도구일 뿐, 결국 기획 의도가 명확해야 좋은 결과가 나온다

### 2. 프롬프트 설계 > 비싼 모델
- 비용 제약(720p, 쇼츠당 2,000원 이하) 안에서도 프롬프트 엔지니어링으로 품질 극대화 가능
- 빛, 구도, 배경, 톤 등 디테일한 프롬프트 설계가 모델 스펙보다 품질에 더 큰 영향

### 3. 데이터 기반 반복 개선
- 시청 유지율 16% → 31.8%는 감이 아니라 YouTube Analytics 데이터를 보며 반복 개선한 결과
- 시니어 채널을 테스트베드로 활용 → 검증된 품질로 바네스데코 협업 진입

### 4. 실제 클라이언트와의 협업
- 개인 프로젝트를 넘어 실제 비즈니스 문제를 해결하는 경험
- 클라이언트 요구사항(완전 무관여, 2,000원 이하, 10개+ 채널)이 기획의 방향을 잡아줌
